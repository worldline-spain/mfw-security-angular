$mfwSecurityRouteInterceptorConfig
object in module mfw.security.route-interceptor.uirouter

Description

Configuration object for $mfwSecurityRouteInterceptor service.

Properties

  • avoidLoginStateWhenLoggedIn

    Default value: false

    Returns
    Boolean

    Whether user should be redirected to the default state when accesses login state being already logged in.

  • defaultState

    Default value: undefined (do nothing after login)

    Returns
    String

    Name of the state to be redirected after login, if any, or after an stopped transition.

  • forwardToDefaultStateAfterLogin

    Default value: false

    Returns
    Boolean

    Whether user should be redirected to the default state once user logs in.

  • forwardToLoginAfterLogout

    Default value: true

    Returns
    Boolean

    Whether user should be redirected to the login state after a successful logout.

  • forwardToLoginAfterStoppedTransition

    Default value: false

    Returns
    Boolean

    Whether user should be redirected to the login state after a stopped transition due to permissions.

  • forwardToRestrictedStateAfterLogin

    Default value: false

    Returns
    Boolean

    Whether user should be redirected to the state that requested login once user logs in.

  • loginState

    Returns
    String

    Flag identifying the login state. More than one will throw an exception. None will cause a warning message.

  • publicByDefault

    Default value: true

    Returns
    Boolean

    Whether all states with no security defined are considered public by default or not.
